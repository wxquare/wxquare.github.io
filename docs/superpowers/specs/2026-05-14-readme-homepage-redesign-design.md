# README Homepage Redesign

## Goal

将仓库根目录 `README.md` 从“项目说明书”重构为“博客首页入口”，优先服务首次访问 GitHub 仓库的读者，让读者在最短时间内理解：

- 这个博客主要写什么。
- 最值得先看的内容是什么。
- 应该从哪个入口开始阅读。

同时保留最基本的本地运行和维护说明，使仓库依然具备可维护性，但不再让开发细节主导首页。

## Chosen Direction

- 主标签：系统设计与后端架构
- 副标签：AI 与 Agent 工程实践
- 信息架构：混合型
- 推荐内容组织方式：精选型

## Target Audience

主要面向两类读者：

1. 第一次访问仓库的技术读者，希望快速判断内容是否有价值。
2. 想基于仓库继续写作或本地运行博客的维护者。

其中第一类为 README 首页的优先对象，第二类放在后半部分满足。

## Problems In Current README

当前 README 的主要问题：

- 首页定位偏弱，只说明“覆盖很多主题”，但没有突出最强内容。
- 快速导航、项目结构、开发规范、工具清单并列出现，信息密度高但优先级不清晰。
- 访客无法快速知道“先看什么”。
- 内容领域展开过长，像目录清单，不像首页导览。
- 写作规范、开发规范、参考资源、链接资源混在同一页，削弱了首页转化。

## Design Principles

- 先回答“值不值得看”，再回答“怎么维护”。
- 先展示代表专题和精选内容，再展示完整内容地图。
- 压缩低价值枚举型内容，保留高价值入口型内容。
- 让系统设计与 AI Agent 两条主线清晰可见。
- README 不是完整知识目录，不追求把所有主题逐项罗列。

## Proposed Structure

### 1. Header

包含标题和一句更强的定位语。

目标：

- 明确这是一个长期维护的技术博客与知识库。
- 突出“系统设计与后端架构”以及“AI 与 Agent 工程实践”。

### 2. Core Entrypoints

保留 3 个核心入口：

- 在线博客
- AI Agent 工程实践
- 系统设计与架构专题

目标：

- 给首次访问者一个直接可点的阅读起点。

### 3. Featured Content

增加“精选内容”区域，采用不分组的精选列表方式。

内容来源以以下两类为主：

- AI 与 Agent 工程实践
- 系统设计与后端架构

目标：

- 快速展示最能代表博客质量和方向的内容。
- 让 README 具备作品集属性。

### 4. Content Map

将“核心内容领域”重写为更短的内容地图，而不是长目录。

建议保留 4 个一级主题：

- 系统设计与后端架构
- AI 与 Agent 工程实践
- 电商架构与性能优化
- 计算机基础

每个主题只保留一句说明或少量关键词，不再展开多层级列表。

### 5. Local Development

保留最基本的运行说明：

- 环境要求
- 安装依赖
- 本地预览
- 构建

`deploy`、大量开发规范、推荐工具不再放在前半部分。

### 6. Repository Layout

保留精简版目录结构说明，只解释与阅读和维护直接相关的目录。

### 7. Writing And Maintenance Notes

把以下内容收敛为“维护附录”：

- Front Matter 约定
- 基本写作规范
- 常见陷阱

开发规范、推荐工具、外部链接资源从 README 首页移除，必要时后续迁移到单独文档。

## Content Removal Strategy

以下内容建议从 README 主体移除或明显压缩：

- 过长的主题枚举
- 推荐工具清单
- 大段开发规范
- 大量外部参考链接

原因：

- 它们不帮助首次访问者判断博客价值。
- 它们会稀释专题入口和精选内容的注意力。

## Writing Style

- 中文为主，术语表达简洁直接。
- 少做“覆盖全面”式空泛描述，多用“你可以看到什么”式表述。
- 控制层级深度，避免三层以上标题堆叠。
- 优先短段落和短列表，减少大块枚举。

## Success Criteria

重写后的 README 应满足：

- 用户在第一屏就能理解博客主线。
- 用户在 10 秒内能找到至少一个阅读入口。
- 首页结构明显从“说明书”转为“内容入口”。
- 维护信息仍然存在，但不会抢占首页注意力。

## Implementation Scope

本次仅调整 `README.md` 的信息架构和文案，不涉及：

- 博客主题样式修改
- 站内导航修改
- 新增文章内容
- 其他目录文档重写

## Open Point

精选内容中的具体文章和专题链接，需要在实际改写 `README.md` 前从现有仓库中筛选确定。
