## 设计方案选择

**选定：方案 1（统一导购查询入口）+ 方案 3（Elasticsearch 技术纵深）合并**

| 维度 | 方案 1 承担 | 方案 3 承担 |
|------|-------------|-------------|
| 叙事主线 | 单一 **导购查询服务**；`scene=keyword \| category \| shop` 共用 **召回 → 排序 → hydrate** | 不作为「替代商品中心索引设计全文」 |
| 技术纵深 | 编排、批量 hydrate、超时、限流、与上下游契约 | **分析链、mapping 与字段策略、分词与同义词、典型 DSL、聚合与筛选、深分页、慢查询与容量、调优清单** |
| 对外 API | 可描述为统一入口或「双 HTTP 路由共享内核」（实现细节二选一，正文只保留一种以免读者混乱） | ES 为召回主存储；降级路径单独一节 |

**未选方案 2 的理由：** 双入口（Search Service + Listing BFF）在面试叙述中增加记忆负担；若需体现拆分，用一节 **「演进：何时拆 BFF」** 作为可选小节即可。

## 与其它系统的交互（集成一章的必选内容）

### 责任边界（摘要）

- **商品中心**：商品主数据与读模型版本；列表 **batch get** 的接口契约（字段裁剪、并发上限、超时）。
- **上架 / listing**：哪些状态 **写入索引 / 更新可搜位 / 物理删文档**；索引滞后与 **延迟可见** 的产品话术与技术对策一致。
- **生命周期 / 审核**：审核通过与拒绝、风控标签对 **召回 filter** 与 **排序降权** 的影响；幂等与重复事件消费。
- **B 端运营**：运营加权、置顶、资源位配置：若来自 **旁路配置服务**，在 **重排** 阶段合并；写清 **更新频率、配置版本与实验桶** 的关联。
- **计价 / 库存 / 营销**：仅写 **读路径** 与 **失败降级**；事件名与 Topic 在成文时用 **grep 仓库** 与 `21`/`27`/`30` 已有命名对齐，本文 spec 不绑定具体字符串以免与代码/文章漂移。

### 建议数据流（正文需配图）

1. 上游领域事件（商品变更、上架状态变更等）→ 索引构建链路（**引用 `27`/`21` 已有描述**）→ ES 文档可见。
2. 用户请求 → Query 理解（轻量）→ ES 召回 → 粗排 / 精排 / 重排（分层写清）→ **Batch hydrate**（价、库存摘要、营销标）→ 响应。
3. 异步：曝光 / 点击写入分析侧（最终一致），用于报表与后续排序迭代（点到为止，不展开数仓）。

## 文章结构（章节级规格）

### 第 1 章：引言与业务背景

- 搜索与结构化导购在 GMV 与体验上的位置；读多写少、与交易读路径的差异。

### 第 2 章：系统定位、范围与非目标

- 显式列出 **A+B** 范围与 **不包含** 列表（与前文「范围外」一致）。
- 与 `27`/`21`/`30`/`25`/`23`/`22`/`28` 的分工表（可浓缩为一张 Markdown 表）。

### 第 3 章：统一导购查询服务（方案 1）

- **场景**：`keyword` / `category`（类目列表）/ `shop`（店铺内）。
- **对外形态**：推荐统一为一种主叙事（例如单服务多 scene）；若提 BFF，放在「演进」小节。
- **主链路**：召回 → 排序分层 → hydrate；各 scene 差异用 **对比表**（是否有 query、固定 filter、索引 alias 等）。

### 第 4 章：Query 理解与召回

- 归一化、同义词 / 词典、可选拼写纠错（**声明复杂度上限**，避免写成 NLP 专著）。
- ES 为主召回；**多路召回（关键词 + 向量）** 仅作 **可选扩展段落**，默认篇幅不超过一节。

### 第 5 章：排序、重排与实验（AB）

- **粗排**（ES score + 简单业务规则）→ **精排**（特征加权，以可解释规则 / 线性加权为主，LTR 一笔带过）→ **重排**（多样性、疲劳度、合规）。
- 实验桶与 **排序配置版本** 的可观测字段（`rank_version`、`exp_id`）。

### 第 6 章：Elasticsearch 专题（方案 3）

- 分析链与中文分词选型原则；**mapping 设计要点**（keyword vs text、`doc_values`、避免过度 nested）。
- **典型查询模式**：筛选 + 排序 + 高亮 + suggest；**聚合与导航属性**。
- **深分页**：`search_after`、禁止高 offset；面试标准表述。
- **性能与容量**：分片副本、慢查询治理、冷热索引一句话、与压测指标占位（成文时填具体数字或保持量级示例）。

### 第 7 章：与其它系统的集成与契约

- 展开「集成」一节：责任表 + **1～2 张序列图**（索引更新路径、列表请求路径）。
- 事件消费语义：**至少一次** 下的幂等（`document_id` + `version` / `updated_at`）在正文写清。

### 第 8 章：一致性、降级与韧性

- 索引滞后；hydrate 部分失败；ES 不可用时的降级策略（缓存快照 vs 受限 DB 查询）**二选一写推荐默认**，另一方案写权衡。
- 限流、防刷触发条件（与网关协同，点到为止）。

### 第 9 章：可观测性与容量

- 指标：零结果率、P99、hydrate 成功率、ES 慢查询、实验分桶对比。
- 日志与 trace：`query_id` 贯穿。

### 第 10 章：工程实践清单

- 发布与索引重建（reindex）风险一句；与 `27` 刷新策略呼应。

### 第 11 章：面试问答锦囊

- 与第 3～8 章一一对应：深分页、相关性 vs 商业化、ES 与 DB 分工、列表价一致性、索引延迟、幂等、压测关注点等。

### 第 12 章：总结与系列索引

- 链接到 `20`～`30` 相关篇；说明下一篇可选方向（如履约 / checkout）不在本文范围。
