## 6. 执行顺序

由于文件重命名存在编号冲突（如原 18 要变成 22，但原 22 还未删除），建议执行顺序：

1. **Phase 1 — 文件重命名**（需注意冲突，使用临时名避免覆盖）
   - 删除原 22（或先备份）
   - 按依赖顺序重命名所有文件
2. **Phase 2 — 通用改动**
   - 批量修改所有电商文章的 Front Matter 和导航块
3. **Phase 3 — 文章 20 内容整合**
   - 从已删除的文章 22 中提取独特内容并入文章 20
   - 精简文章 20 中的商品管理和稳定性章节
4. **Phase 4 — 文章 25 去重**
   - 检查并处理与文章 20 的重复内容
5. **Phase 5 — 构建验证**
   - 运行 `npm run clean && npm run build`
   - 验证所有交叉引用链接正确

## 7. 风险与注意事项

- **URL 变更**：所有重命名文件的 URL 都会改变，旧链接将 404
- **文件重命名冲突**：多个文件编号互换，需使用临时名中转，避免覆盖
- **文章 20 内容整合**：从文章 22 提取内容时需人工判断取舍，避免引入新的重复
- **构建验证**：每个 Phase 完成后都应执行 `npm run clean && npm run build`
