## 设计方案选择

**选定：分域叙事（购物车域 + 结算域）+ Saga 编排为主线**

| 维度 | 购物车域 | 结算域（Checkout） |
|------|----------|-------------------|
| 叙事主线 | 暂存与展示；匿名与登录态合并；弱一致 | 试算 → 预占 → 校验 → 提交订单；Saga 补偿；强一致 |
| 与订单关系 | 独立生命周期（可长期保留） | 订单前置状态（用完即焚或短暂会话） |
| 技术纵深 | Redis + DB 双写；批量操作幂等；跨端同步 | Saga 模式、`idempotency_key`、超时释放、拆单编排 |

**未选方案**："购物车 = 订单草稿"统一建模：会导致订单表污染与状态机复杂化；推荐保持购物车与订单的边界清晰。

## 与其它系统的交互（集成一章的必选内容）

### 责任边界（摘要）

- **购物车**：暂存商品、聚合展示、批量管理；**不锁定任何资源**（不扣库存、不锁价、不扣券）。
- **结算页**：价格最终确认、资源预占（库存 15 分钟）、拆单预览、提交订单前置校验。
- **订单系统**：接收结算页传入的 `price_snapshot_id`、`reserve_ids`，执行"从预占转确认"；真正拆单与履约路由。

### 建议数据流（正文需配图）

1. **购物车流程**：用户加购 → 匿名 Token 或 `user_id` → Redis/DB → 登录后合并 → 展示（查商品/价格/库存状态）。
2. **结算页编排**：进入结算 → 试算价格（不锁定）→ 预占库存（15 分钟）→ 校验优惠（不扣券）→ 拆单预览 → 用户提交订单（`idempotency_key`）→ 创建订单（确认预占 + 扣券）→ 跳转支付。
3. **补偿路径**：用户放弃结算 → 预占超时自动释放；订单创建失败 → 显式释放预占 + 返回购物车。

## 文章结构（章节级规格）

### 第 1 章：引言与业务背景

- 购物车与结算在转化漏斗的位置（浏览 → 加购 → 结算 → 下单 → 支付）。
- 读多写少（购物车展示）vs 强校验（结算页）的矛盾。

### 第 2 章：系统定位、范围与非目标

- 显式列出 **A+B** 范围与 **不包含** 列表（与前文「范围外」一致）。
- 与 `26`/`23`/`22`/`28`/`27`/`29` 的分工表（可浓缩为一张 Markdown 表）。

### 第 3 章：核心场景与挑战

- **购物车场景**：未登录加购、登录后合并、商品失效（下架/涨价/无货）、批量操作、跨端同步。
- **结算页场景**：价格试算、库存预占、营销校验、拆单编排、地址 & 运费、幂等与重试。
- 挑战对比表（技术挑战 vs 方案）。

### 第 4 章：购物车设计

- **数据模型**：`shopping_cart` 表（`user_id` / `cart_token` / `sku_id` / `quantity` / `selected`）。
- **存储选型**：Redis（HASH 结构）+ DB 双写；匿名与登录态合并策略。
- **批量操作幂等**：修改数量用乐观锁版本号。
- **商品失效展示**：购物车只读标记，不自动删除。

### 第 5 章：结算页设计（Checkout Orchestrator）

- **编排流程**：试算 → 预占 → 校验 → 提交订单；Saga 模式。
- **幂等与去重**：`idempotency_key`（前端生成 UUID）+ Redis `SET NX` + 订单表唯一索引。
- **补偿路径**：预占超时自动释放、订单创建失败显式释放。
- **结算会话表**（可选）：支持断点续结算的权衡（有状态 vs 无状态）。

### 第 6 章：拆单与地址运费

- **拆单策略**：跨店铺、跨仓、自营 + POP；结算页 **预览拆单结果**，真正拆单在订单创建时。
- **地址服务集成**：多地址切换、默认地址、收货人信息校验。
- **运费规则引擎**：运费实时计算（可缓存短时，秒级 TTL）。

### 第 7 章：与其他系统的集成与契约

- 展开「集成」一节：责任表 + **2 张序列图**（购物车查商品、结算页 Saga 编排）。
- 事件消费语义（可选）：订单创建成功后 **清理购物车**（通过消息队列异步，非强依赖）。

### 第 8 章：一致性、降级与韧性

- **购物车**：弱一致（展示可滞后）；商品失效标记而不删除。
- **结算页**：强一致（价格/库存/优惠实时校验）；部分失败降级为"无法结算，请稍后重试"。
- **预占超时释放**：15 分钟过期，定时任务或 TTL 机制。

### 第 9 章：可观测性与转化漏斗

- 指标：加购率、进入结算率、提交订单成功率；漏斗分析定位卡点。
- 日志与 trace：`cart_id`、`checkout_session_id`、`idempotency_key` 贯穿。

### 第 10 章：工程实践清单

- 购物车同步策略（Redis 与 DB 的延迟与补偿）。
- 结算页超时配置（试算 / 预占 / 校验独立超时）。
- 预占释放监控（释放任务执行率、过期预占计数）。

### 第 11 章：面试问答锦囊

- 与第 3～8 章一一对应：未登录加购、合并策略、预占库存、幂等、拆单时机、Saga 补偿、漏斗分析等。
