AI工程实践：从编程到Agent的完整指南

书籍介绍

欢迎阅读《AI工程实践：从编程到Agent的完整指南》！

关于本书

本书面向有经验的AI工程师，系统梳理了AI工程实践的完整方法论，从AI编程工具的使用，到Agent系统的架构设计，再到生产级案例的落地实践。

你将读到什么

第一部分：AI编程实战（3章）

深入探讨AI编程的范式演进，从Vibe Coding到Spec Coding的跃迁，再到Harness Engineering的系统化方法论。涵盖Cursor IDE和Claude Code两大主流工具的完整实践。

核心价值：

  • 系统化的AI编程方法论
  • Cursor和Claude Code完整工作流
  • Harness Engineering设计清单
  • 从即兴编程到规范驱动的转变

第二部分：AI Agent系统设计（4章）

从架构设计到工具系统，从多Agent协作到可观测性，全面覆盖生产级Agent系统的设计与实现。

核心价值：

  • 完整的Agent架构设计方法论
  • ReACT、Plan-and-Execute等核心模式
  • MCP协议与工具系统设计
  • 生产环境的可观测性与成本优化

第三部分：完整实战案例（2章）

通过DoD Agent（电商告警处理系统）和个人知识管理Agent两个完整案例，展示从需求分析到部署运维的全流程实践。

核心价值：

  • 真实的生产级案例
  • 完整的设计决策过程
  • 可复用的架构模式
  • 实战中的坑与解决方案

第四部分：基础理论补充（2章）

精简的LLM和RAG基础理论，作为前面实战内容的理论支撑。

本书特色

1. 实战为主，理论为辅

  • 70%的内容来自真实项目实践
  • 完整的代码示例和架构图
  • 可直接复用的设计模式

2. 系统化方法论

不只是工具使用指南，更是完整的工程方法论：

  • Vibe Coding → Spec Coding → Harness Engineering
  • 单Agent → 多Agent → Agent系统
  • 原型 → 生产 → 优化

3. 工具链完整覆盖

  • AI编程：Cursor、Claude Code
  • Agent框架：LangChain、OpenClaw
  • 协议标准：MCP（Model Context Protocol）
  • 工程实践：RAG、可观测性、成本优化

如何阅读本书

快速上手（2-3天）

如果你需要快速掌握核心内容：

  1. 阅读第1章（AI编程范式）
  2. 选读第2或第3章（工具实践）
  3. 阅读第8章（DoD Agent案例）

系统学习（1-2周）

如果你想系统掌握AI工程实践：

  1. 按顺序阅读第一部分（AI编程实战）
  2. 按顺序阅读第二部分（Agent系统设计）
  3. 选择感兴趣的案例深入学习（第三部分）
  4. 查阅第四部分补充基础理论

案例驱动（自定进度）

如果你有具体的项目需求：

  1. 先看第8或第9章的完整案例
  2. 遇到不理解的概念，回查对应章节
  3. 参考附录C的工具与框架清单

前置知识

本书假设你已经具备：

  • 基本的编程经验（任何语言均可）
  • 对AI/LLM有基本了解
  • 有一定的系统设计经验（后端开发经验加分）
