# /publish - 一键发布博客

执行完整的发布流程，从检查到部署。

## 执行步骤

### 1. 检查Git状态

```bash
git status
```

查看当前有哪些文件被修改。

### 2. 运行构建测试

```bash
npm run clean && npm run build
```

清理缓存并重新构建，确保没有错误。

### 3. 查看变更

```bash
git diff --stat
```

查看文件变更统计。

### 4. 添加变更到暂存区

询问用户：
- 是否添加所有变更？(y/n)
- 或指定要添加的文件

```bash
git add .
# 或
git add [指定文件]
```

### 5. 生成提交信息

分析变更内容，生成有意义的commit message：
- 如果是新增文章：`feat: add [文章标题]`
- 如果是更新文章：`update: [文章标题]`
- 如果是修复：`fix: [问题描述]`
- 如果是多个变更：列出主要变更

### 6. 提交变更

```bash
git commit -m "[生成的提交信息]"
```

### 7. 推送到远程

```bash
git push origin hexo
```

### 8. 部署到GitHub Pages

```bash
npm run deploy
```

### 9. 验证部署

- 显示博客URL
- 建议等待1-2分钟让GitHub Pages更新
- 提示检查部署是否成功

## 可选步骤

### 创建Git Tag（重要更新）

询问用户是否需要创建tag：
```bash
git tag -a v[版本号] -m "[说明]"
git push origin v[版本号]
```

### 生成变更日志

如果有多个提交，生成变更日志：
```bash
git log --oneline --since="1 week ago"
```

## 安全检查

发布前会自动检查：
- [ ] Front Matter完整性
- [ ] 构建是否成功
- [ ] 是否有未追踪的重要文件
- [ ] 是否有敏感信息（如API密钥）

如果发现问题，会提示用户确认是否继续。

## 回滚机制

如果部署失败，提供回滚选项：
```bash
git reset --soft HEAD~1  # 撤销提交但保留更改
```

## 使用场景

- 新文章发布
- 文章更新
- 批量发布多篇文章
- 紧急修复
