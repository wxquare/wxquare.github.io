# /link-check - 检查文章中的链接

扫描所有文章中的链接，检查有效性，生成失效链接报告。

## 功能

### 1. 扫描链接
- 提取所有Markdown链接 `[text](url)`
- 提取所有HTML链接 `<a href="url">`
- 提取图片链接 `![alt](url)`
- 提取参考链接 `[text][ref]`

### 2. 分类检查

**内部链接**
- 检查文件是否存在
- 检查路径是否正确
- 检查锚点是否有效

**外部链接**
- 检查URL是否可访问（HTTP状态码）
- 检查是否重定向
- 检查响应时间

**图片链接**
- 检查图片文件是否存在
- 检查图片格式是否支持
- 检查图片大小是否合理

### 3. 生成报告
- 失效链接列表
- 慢速链接列表
- 重定向链接列表
- 修复建议

## 执行步骤

1. **扫描所有文章**
```bash
find source/_posts -name "*.md" -type f
```

2. **提取链接**
   - 使用正则表达式提取
   - 分类整理

3. **并发检查**
   - 内部链接：文件系统检查
   - 外部链接：HTTP请求（限制并发数）
   - 超时设置：10秒

4. **生成报告**

## 检查规则

### 内部链接
```
✅ 正确：../other/article.md
✅ 正确：/images/diagram.png
❌ 错误：/absolute/path/article.md
❌ 错误：article.md（相对路径不明确）
```

### 外部链接
```
✅ 200 OK
⚠️ 301/302 重定向（建议更新）
❌ 404 Not Found
❌ 超时
❌ 连接失败
```

### 图片链接
```
✅ 本地图片存在
✅ 外部图片可访问
⚠️ 图片过大（>1MB）
❌ 图片不存在
❌ 格式不支持
```

## 使用场景

- 发布前检查
- 定期维护（建议每月一次）
- 更新旧文章前
- 迁移文章后

## 配置选项

```yaml
# .claude/link-check-config.yml
timeout: 10  # 超时时间（秒）
concurrent: 5  # 并发请求数
retry: 2  # 重试次数
ignore_patterns:  # 忽略的URL模式
  - "localhost"
  - "127.0.0.1"
  - "example.com"
```

## 注意事项

- 外部链接检查会发送HTTP请求，注意频率限制
- 某些网站可能阻止爬虫，导致误报
- 检查大量链接需要时间，建议异步执行
- 定期检查比一次性检查更有效
