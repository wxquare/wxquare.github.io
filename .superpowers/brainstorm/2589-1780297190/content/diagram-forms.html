细化：分层图的具体布局风格

既然选择「纵向分层图」，接下来决定具体的布局方式

A

标准 TB - 纯纵向堆叠

┌─ 用户 ────┐
  商家、BD、供应链
└───────────┘
      │
┌──────▼───────┐
│ 商品中心(桥梁) │
└──────┬───────┘
      │
┌──▼──┐  ┌──▼──┐
│供给  │  │运营  │
└──┬───┘  └──┬──┘
      │      │
    ┌─▼──────▼─┐
    │  消费者    │
    └──────────┘

供给和运营在桥梁两侧并排

B

用户在上 + 平台左右对置

┌─ 供给用户 ─┐   ┌─ 运营用户 ─┐
├─ 商家/BD  ─┤   ├─ 品类/活动 ─┤
└──────┬─────┘   └──────┬──────┘
       │                │
┌──────▼─────────────────▼──────┐
│        商品中心(Bridge)        │
├───────────────────────────────┤
│  SPU/SKU · 合规 · 版本 · 发布 │
└──────┬─────────────────┬──────┘
       │                │
┌──────▼────┐   ┌──────▼──────┐
│ 供给平台    │   │  运营平台    │
│ 入驻/供应链│   │  营销/CMS  │
└──────┬─────┘   └──────┬──────┘
       │                │
       └───────┬────────┘
               ▼
          ┌─────────┐
          │ 消费者    │
          └─────────┘

用户层分开，显示两个平台面向不同群体

C

完整四层架构

Layer 1:  入口层 ──  商家Portal / 运营工作台 (各自独立)
                    👇
Layer 2:  业务层 ──  供给系统群 | 运营系统群
                    👇
Layer 3:  数据层 ──  商品中心(主数据) | 库存系统 | 定价系统
                    👇
Layer 4:  触达层 ──  搜索 · 推荐 · 促销 · 消费者
