# 第 20 章 动态规划速查

> 本部分属于编码补充，用来和系统设计主线互补；更适合作为查漏补缺与面试前速查，不要求线性通读。

## DP 的核心思想

动态规划的本质是“把重复子问题的答案保存下来”，难点不在代码模板，而在状态设计。

**五个固定问题**

1. `dp` 数组或函数的定义是什么？
2. 状态如何转移？
3. 初始条件和边界条件是什么？
4. 计算顺序是什么？
5. 最终答案落在哪个状态上？

**两种常见视角**

- 自底向上填表：依赖关系清晰、便于做空间优化。
- 自顶向下记忆化：问题表达自然、适合先写递归再补缓存。

**两类依赖模式**

- 局部依赖：`dp[i]` 只看前一两个状态，如 `70. 爬楼梯`、`198. 打家劫舍`、`53. 最大子数组和`。
- 全局扫描：`dp[i]` 需要枚举所有历史状态，如 `300. 最长递增子序列`、`139. 单词拆分`。

把题看成“状态 + 转移图”，比死记“这是哪一类 DP”更可靠。

## 线性 DP 与背包问题

**线性 DP**

- 斐波那契模型：`509. 斐波那契数`、`70. 爬楼梯`、`746. 使用最小花费爬楼梯`。
- 网格路径：`62. 不同路径`、`63. 不同路径 II`、`64. 最小路径和`、`120. 三角形最小路径和`。
- 状态机：`121. 买卖股票的最佳时机`、`122. 买卖股票的最佳时机 II`、`309. 最佳买卖股票时机含冷冻期`。
- 双串与序列：`1143. 最长公共子序列`、`72. 编辑距离`、`97. 交错字符串`、`115. 不同的子序列`。
- 划分型：`132. 分割回文串 II`、`1043. 分隔数组以得到最大和`、`410. 分割数组的最大值`。

**背包问题**

- 0/1 背包：`416. 分割等和子集`、`494. 目标和`、`474. 一和零`。
- 完全背包：`322. 零钱兑换`、`518. 零钱兑换 II`、`279. 完全平方数`、`377. 组合总和 IV`。
- 分组背包：`1155. 掷骰子等于目标和的方法数`。

**最重要的实现差异**

- 0/1 背包一维优化时，容量要倒序遍历。
- 完全背包一维优化时，容量要正序遍历。
- 题目问“最值”“是否可行”“方案数”时，转移写法都不同，不能混套模板。

## 区间 DP、树形 DP、状态压缩 DP

**区间 DP**

- 适合“合并区间”“最后一步切分”的题。
- 代表题：`312. 戳气球`、`877. 石子游戏`、`375. 猜数字大小 II`。
- 典型写法是枚举区间长度，再枚举切分点。

**树形 DP**

- 节点状态依赖子树返回值，适合树上最优解与路径聚合。
- 代表题：`543. 二叉树的直径`、`124. 二叉树中的最大路径和`、`337. 打家劫舍 III`、`834. 树中距离之和`。
- 难点往往不是遍历，而是定义“每个节点返回什么信息”。

**状态压缩 DP**

- 当元素数量很小但子集状态很多时，把集合压成二进制。
- 代表题：`464. 我能赢吗`、`526. 优美的排列`、`698. 划分为 k 个相等的子集`、`2741. 特别的排列`。
- 常和记忆化搜索、位运算一起出现。

## 常见套路总结

| 信号 | DP 类型 | 代表题 |
| --- | --- | --- |
| 只依赖前几个位置 | 线性 DP | `70`、`198`、`746` |
| 二维网格或双串匹配 | 二维 DP | `64`、`1143`、`72` |
| 买卖股票、持有/不持有 | 状态机 DP | `121`、`309` |
| 按区间合并 | 区间 DP | `312`、`375` |
| 子树向上汇总 | 树形 DP | `124`、`337` |
| 小规模集合枚举 | 状压 DP | `464`、`526` |
| 容量限制选物品 | 背包 DP | `416`、`322`、`518` |

复习时不要只背题型名，更要养成这几个习惯：

- 先写出状态含义，再写转移。
- 先画依赖方向，再决定循环顺序。
- 空间优化前先确认不会覆盖掉本轮还需要的旧状态。

## 本章面试题与追问

- 为什么说“状态定义正确，DP 就做完一半了”？你如何检查状态是否完备？
- 0/1 背包为什么必须倒序遍历，正序会错在哪里？
- 区间 DP 常见的枚举顺序为什么是“先长度、后左端点、再分割点”？
- 记忆化搜索和 DP 填表本质上有什么关系，什么时候更适合前者？
- 股票状态机题里，如何把“是否持有、是否冷冻、已经做过几次交易”编码成状态？
