# 第 45 章 数据结构与算法题型速查

> 本章属于编码补充，用来和系统设计主线互补；更适合作为查漏补缺与面试前速查，不要求线性通读。

## 数据结构题型

### 数组、链表、栈与队列

这类题目的共同点是数据组织方式明确，重点在于识别访问顺序、边界移动方式，以及是否允许原地修改。

**数组与矩阵**

- 原地置换与原地标记：`41. 缺失的第一个正数`、`448. 找到所有数组中消失的数字`、`73. 矩阵置零`。
- 坐标变换与遍历模拟：`48. 旋转图像`、`54. 螺旋矩阵`、`36. 有效的数独`、`31. 下一个排列`。
- 区间与贡献度：`53. 最大子数组和`、`152. 乘积最大子数组`、`907. 子数组的最小值之和`。

**链表**

- 基础反转与局部反转：`206. 反转链表`、`92. 反转链表 II`、`25. K 个一组翻转链表`。
- 快慢指针与结构定位：`141. 环形链表`、`142. 环形链表 II`、`19. 删除链表的倒数第 N 个结点`、`876. 链表的中间结点`。
- 合并、排序与拆分：`21. 合并两个有序链表`、`23. 合并 K 个升序链表`、`148. 排序链表`、`86. 分隔链表`。
- 复杂结构与缓存设计：`138. 随机链表的复制`、`382. 链表随机节点`、`146. LRU 缓存`、`460. LFU 缓存`。

**栈与队列**

- 基础栈模型：括号匹配、表达式处理、递归转迭代。
- 单调栈：`84. 柱状图中最大的矩形`、`496. 下一个更大元素 I`、`739. 每日温度`、`42. 接雨水`、`2866. 美丽塔 II`。
- 双端队列与窗口：`239. 滑动窗口最大值`、`862. 和至少为 K 的最短子数组`、`2810. 故障键盘`。
- 队列分层处理：树的层序遍历、图的 BFS、任务调度等题型。

**做题时先问自己**

- 数据是否要求连续存储，能否通过下标直接映射状态？
- 是否必须保留原顺序，还是只关心相对顺序或分组结果？
- 是否存在“最近更大/更小”“窗口最值”“前驱/后继”这类单调性信号？
- 是否可以通过哑节点、辅助栈、队首队尾维护，把边界操作写干净？

### 哈希表、堆、树与图

这一组题的核心不是“存数据”，而是“如何高效定位、聚合与搜索”。

**哈希表**

- 频次统计与去重：`1. 两数之和`、`49. 字母异位词分组`、`128. 最长连续序列`。
- 前缀和配合哈希：`560. 和为 K 的子数组`、`437. 路径总和 III`。
- 映射复杂对象：克隆图、随机链表复制、缓存设计等。
- 典型信号：题目要求均摊 `O(1)` 查找，或反复判断“是否见过”“出现次数是多少”。

**堆与优先队列**

- Top K 与动态极值：`215. 数组中的第 K 个最大元素`、`347. 前 K 个高频元素`、`295. 数据流的中位数`。
- 多路归并：`23. 合并 K 个升序链表`、`264. 丑数 II`。
- 反悔贪心：`630. 课程表 III`、`871. 最低加油次数`、`502. IPO`。
- 图最短路：`743. 网络延迟时间`、`1976. 到达目的地的方案数`。

**树**

- 遍历与模板：前序、中序、后序、层序遍历。
- 路径与聚合：`112. 路径总和`、`113. 路径总和 II`、`124. 二叉树中的最大路径和`、`543. 二叉树的直径`。
- 二叉搜索树性质：中序有序、范围裁剪、前驱后继。
- Trie 与字典树：`208. 实现 Trie`、`212. 单词搜索 II`，适合多模式前缀匹配。

**图**

- 连通性与遍历：`200. 岛屿数量`、`133. 克隆图`、`2101. 引爆最多的炸弹`。
- 拓扑排序：`207. 课程表`、`210. 课程表 II`。
- 并查集：`547. 省份数量`、`684. 冗余连接`、`1998. 数组的最大公因数排序`。
- 最短路与状态图：`127. 单词接龙`、`407. 接雨水 II`、`2642. 设计图类求最短路径`。

**选择提示**

- “找是否存在、统计频率”优先看哈希。
- “维护动态最小值/最大值/前 K”优先看堆。
- “天然层级结构、父子依赖”优先看树。
- “对象之间有边、有依赖、有可达性”优先建图。

### 常见题型总结

这一章更像一个数据结构选型速查表：

| 信号 | 常用结构 | 代表题 |
| --- | --- | --- |
| 原地修改、数组下标可映射状态 | 数组原地哈希 | `41`、`448`、`73` |
| 最近更大/更小、区间贡献 | 单调栈 | `84`、`739`、`907` |
| 窗口最值、可弹出过期元素 | 双端队列 | `239`、`862` |
| 快速判重、计数、前缀信息 | 哈希表 | `1`、`49`、`560` |
| Top K、动态极值、反悔 | 堆 | `215`、`295`、`630` |
| 指针操作复杂、局部重连 | 链表 + 哑节点 | `19`、`25`、`86` |
| 层级遍历、路径聚合 | 树 | `102`、`124`、`543` |
| 连通性、依赖、最短路 | 图 / 并查集 | `200`、`207`、`743` |

复习时建议按“结构能力”而不是按题号记忆：

- 数组看原地性、连续性、前后缀。
- 链表看指针关系、是否需要前驱、是否适合哑节点。
- 栈和队列看单调性与过期元素处理。
- 树和图看遍历顺序、状态定义和去重方式。

- 为什么 `LRU` 需要“哈希表 + 双向链表”，只用其中一种结构为什么不够？
- 单调栈题里，为什么元素出栈时往往意味着“答案已经确定”？
- 用快慢指针找环入口时，为什么相遇后一个指针回到头节点再同步前进即可？
- `Top K` 题什么时候用堆，什么时候用快速选择，什么时候用桶排序？
- DFS、BFS、并查集都能做连通性问题时，面试里该如何解释选型差异？

## 基本算法模式

### 双指针、滑动窗口与前缀和

这一组方法都在处理“连续区间”，区别在于维护的信息不同。

**双指针**

- 对撞指针：适合有序数组与区间边界博弈，代表题有 `167. 两数之和 II`、`15. 三数之和`、`11. 盛最多水的容器`、`42. 接雨水`。
- 快慢指针：适合原地压缩与稳定搬运，代表题有 `283. 移动零`、`392. 判断子序列`、`151. 反转字符串中的单词`。
- 中心扩展也可视作双指针特例，常见于 `5. 最长回文子串`、`647. 回文子串`。

**滑动窗口**

- 求最长或最大：窗口合法时持续右扩，例如 `3. 无重复字符的最长子串`、`1004. 最大连续 1 的个数 III`、`2024. 考试的最大困扰度`。
- 求最短或最小：一旦满足条件就尽量收缩左边界，例如 `76. 最小覆盖子串`、`209. 长度最小的子数组`。
- 固定窗口：左右同步移动，更适合做统计量维护，例如 `438. 找到字符串中所有字母异位词`、`1052. 爱生气的书店老板`。

**前缀和**

- 区间和直接查询：`sum[l..r] = pre[r+1] - pre[l]`。
- 配合哈希做计数：`560. 和为 K 的子数组`。
- 配合单调队列解决负数场景：`862. 和至少为 K 的最短子数组`。
- 二维前缀和适合矩阵区域统计，常见于子矩阵和类问题。

**判断标准**

- 区间连续且能在线维护条件，优先考虑窗口。
- 输入有序或能通过排序制造单调性，优先考虑双指针。
- 反复查询区间和、区间计数、区间差值，优先考虑前缀和。

### 二分、排序与贪心

这三类方法都依赖“结构化信息”，只是利用方式不同。

**二分**

- 标准查找：`704. 二分查找`、`34. 在排序数组中查找元素的第一个和最后一个位置`。
- 在旋转数组中找局部单调性：`33. 搜索旋转排序数组`、`153. 寻找旋转排序数组中的最小值`。
- 二分答案：把“能否做到”转成判定函数，例如 `875. 爱吃香蕉的珂珂`、`1011. 在 D 天内送达包裹的能力`、`410. 分割数组的最大值`。

**排序**

- 排序后消除一个维度：`56. 合并区间`、`252. 会议室`、`179. 最大数`。
- 自定义排序：`49. 字母异位词分组`、`179. 最大数`、`2512. 奖励最顶尖的 K 名学生`。
- 排序 + 归并思想：`315. 计算右侧小于当前元素的个数`、`493. 翻转对`。

**贪心**

- 局部最优可推出全局最优：`455. 分发饼干`、`134. 加油站`、`135. 分发糖果`。
- 反悔贪心：`630. 课程表 III`、`502. IPO`、`871. 最低加油次数`。
- 贡献度视角：`2171. 拿出最少数目的魔法豆`、`2731. 移动机器人`。

**面试表达建议**

- 先说明问题是否具备单调性。
- 再说明排序是否改变答案本质，只是帮助更快做决策。
- 最后解释贪心选择为何不会错过最优解，必要时给反证或交换论证。

### 递归与回溯基础

递归适合表达“问题拆成同类子问题”，回溯是在递归基础上显式撤销选择。

**递归框架**

- 明确函数含义、终止条件、子问题返回值。
- 树题常用“返回子树信息”的递归。
- 分治题常用“拆分后合并”的递归，如归并排序、快速幂、快速选择。

**回溯框架**

- 路径 `path`：当前已经做出的选择。
- 选择列表：当前位置还能尝试什么。
- 结束条件：何时收集答案或提前返回。
- 撤销选择：恢复现场，避免污染其他分支。

**三类高频模型**

- 组合：用 `start` 控制“之后还能选谁”，代表题 `77. 组合`、`39. 组合总和`、`216. 组合总和 III`。
- 排列：用 `used` 控制“哪些元素已经用过”，代表题 `46. 全排列`、`47. 全排列 II`。
- 分割：枚举切分点并维护合法性，代表题 `131. 分割回文串`、`LCR 086. 分割回文串`。

**常见剪枝**

- 排序后剪枝：重复元素跳过、超过目标直接终止。
- 上下界剪枝：剩余元素数量不足，或当前和已经不可能达到目标。
- 记忆化：相同状态重复访问时直接复用结果。

### 常见题型总结

| 信号 | 方法 | 代表题 |
| --- | --- | --- |
| 连续区间、窗口可维护 | 滑动窗口 | `3`、`76`、`209` |
| 有序数组、两端逼近 | 双指针 | `167`、`15`、`11` |
| 高频区间统计 | 前缀和 | `560`、`862` |
| 答案具有单调性 | 二分答案 | `875`、`1011`、`410` |
| 先排序再做决策更自然 | 排序 + 贪心 | `56`、`252`、`455` |
| 枚举所有方案 | 回溯 | `39`、`46`、`131` |

把这些方法连起来理解会更稳：

- 双指针与窗口本质上都在维护一个区间。
- 前缀和负责把“区间信息”提取成常数时间查询。
- 二分和贪心通常依赖排序或单调性来缩小搜索空间。
- 回溯负责“全搜”，剪枝负责“少搜”。

### 本章面试题与追问

- 滑动窗口和双指针的边界区别是什么，为什么两者经常被混用？
- 二分答案题里，如何快速判断题目是否满足“可行解单调”？
- 贪心题中如果无法严格证明，面试时该如何稳妥表述？
- 回溯去重时，为什么常见写法是“排序后跳过同层重复元素”？
- `560` 和 `862` 都和前缀和有关，为什么一个用哈希，一个要加单调队列？

## 数学与位运算

### 数论与位运算

这类题常见于“公式不复杂，但实现细节容易错”的场景，核心是把数学性质翻译成稳定代码。

**数论基础**

- 质数与筛法：`204. 计数质数`，掌握试除、埃氏筛与线性筛的复杂度差异。
- 因子统计：`172. 阶乘后的零`、`263. 丑数`，本质是统计某类质因子的幂次。
- 最大公约数与最小公倍数：`1998. 数组的最大公因数排序`，常和并查集、连通性一起出现。
- 数字处理：`9. 回文数`、`400. 第 N 位数字`、`343. 整数拆分`。
- 多数投票：`169. 多数元素`、`229. 多数元素 II`，要会解释 Boyer-Moore 的“抵消”直觉。

**位运算**

- 基础恒等式：`n & (n - 1)` 用来消掉最低位 `1`，`n & -n` 用来提取 lowbit。
- 位计数与位逆序：`191. 位 1 的个数`、`190. 颠倒二进制位`。
- 幂与公共前缀：`231. 2 的幂`、`201. 数字范围按位与`。
- 异或专题：`136. 只出现一次的数字`、`137. 只出现一次的数字 II`、`260. 只出现一次的数字 III`、`421. 数组中两个数的最大异或值`。

**常见组合方式**

- 数论 + 并查集：通过公因数连边。
- 位运算 + 状态压缩：把集合状态压成整数。
- 异或 + 分组：用 lowbit 把两个答案拆到不同桶里。

### 概率、组合与常见技巧

这部分更像工具箱，很多题都不是硬背公式，而是识别“应当用哪种计数或随机模型”。

**组合计数**

- 排列组合基础：理解 `C(n, k)` 与排列数的差别。
- 典型题：`3405. 统计恰好有 K 个相等相邻元素的数组数目`、`3154. 到达第 K 级台阶的方案数`。
- 题目如果要求“方案数”且顺序是否重要不明确，先判断是组合还是排列。

**模运算**

- 常见写法：加法、减法、乘法都要及时取模，避免中间值溢出。
- 除法不能直接取模，通常要转成乘以逆元。
- 代表题：`3463. 判断操作后字符串中的数字是否相等 II`、`3790. 最小全 1 倍数`。

**概率与随机**

- 水塘抽样：`398. 随机数索引`，适合流式输入与未知长度序列。
- Fisher-Yates 洗牌：`384. 打乱数组`，要会解释“每个位置等概率落到任意元素”。
- 拒绝采样：`470. 用 Rand7() 实现 Rand10()`，重点是构造等概率样本空间。

**快速幂与几何**

- 快速幂：`50. Pow(x, n)`，是数论、矩阵快速幂、逆元计算的基础。
- 几何归类：`149. 直线上最多的点数`、`3102. 最小化曼哈顿距离`，重点在坐标变换与斜率规范化。

### 易错点总结

- 组合数与排列数混淆，导致把“顺序不同算不同方案”写错。
- 模运算里先除后模，或在非质数模数下错误使用费马小定理。
- 用 `int` 存中间乘积，结果在取模前已经溢出。
- 处理负数取模时忘记归一化到非负范围。
- 位运算中忽略有符号右移、补码表示和语言差异。
- `rand` 题只写出思路，没说明为什么“等概率”成立。
- 几何题用浮点数直接比斜率，导致精度问题；更稳妥的做法是约分成整数对。

## 搜索问题

### DFS 与 BFS

搜索题的第一步不是写代码，而是先定义“状态”和“边”。

**BFS**

- 适合无权图最短路、分层扩展、最少步数问题。
- 代表题：`200. 岛屿数量`、`909. 蛇梯棋`、`127. 单词接龙`。
- 多源 BFS：`2812. 找出最安全路径`、`2258. 逃离火灾`，先把多个起点同时入队。
- 双向 BFS：起点和终点都明确、状态可逆时收益明显，如 `127. 单词接龙`。

**DFS**

- 适合连通性判断、遍历整张图、做树上聚合。
- 代表题：`200. 岛屿数量`、`133. 克隆图`、`2101. 引爆最多的炸弹`。
- 树上 DFS：`112. 路径总和`、`113. 路径总和 II`、`2477. 到达首都的最少油耗`。
- 记忆化 DFS：当状态会重复访问时，搜索可以自然过渡成 DP。

**如何选**

- 题目问“最少步数”或“最短层数”，优先 BFS。
- 题目问“是否可达”“有多少连通块”“收集所有方案”，优先 DFS。
- 如果状态很多但存在重叠，考虑记忆化搜索。

### 回溯与剪枝

回溯本质上是在状态树上做 DFS，只不过我们更关心“生成所有合法解”。

**基础模型**

- 组合：`77. 组合`、`39. 组合总和`、`40. 组合总和 II`。
- 排列：`46. 全排列`、`47. 全排列 II`。
- 分割：`131. 分割回文串`、`LCR 086. 分割回文串`。
- 约束满足：`51. N 皇后`、`22. 括号生成`、`79. 单词搜索`、`212. 单词搜索 II`。

**剪枝思路**

- 排序后提前终止：例如当前和已经超过目标值。
- 去重剪枝：同层遇到相同值，只保留第一个分支。
- 可行性剪枝：剩余元素数量不足，或约束已经被破坏。
- 结构剪枝：Trie、位掩码、列/对角线标记等，用数据结构减少无效搜索。

**常用表达**

- “做选择”
- “递归进入下一层”
- “撤销选择”

这三句话如果能和具体状态变量对应起来，回溯题就不容易写乱。

### 状态搜索与图搜索

很多难题并不是普通遍历，而是“在隐式图上搜索状态”。

**状态设计**

- 状态要能唯一描述当前位置与历史影响，例如 `(node, mask)`、`(x, y, keys)`。
- 去重依据必须和状态定义一致，否则会误剪枝或重复搜索。

**图搜索常见模式**

- Dijkstra / 优先队列 BFS：`407. 接雨水 II`、`743. 网络延迟时间`、`2045. 到达目的地的第二短时间`。
- 状压搜索：`847. 访问所有节点的最短路径`、`464. 我能赢吗`、`2850. 将石头分散到网格图的最少移动次数`。
- 逆向搜索：`130. 被围绕的区域`，从“不能被包围”的边界反向标记更自然。
- 启发式搜索：如八数码的 `A*`，用于状态空间极大但存在可行估价函数的情况。

**实战建议**

- 先判断图是否有权，边权是否都相等。
- 再判断状态是否需要携带额外历史信息。
- 最后决定用队列、栈、优先队列还是递归栈实现。

### 常见题型总结

| 问题类型 | 常用方法 | 代表题 |
| --- | --- | --- |
| 无权最短路 | BFS | `127`、`909` |
| 连通块统计 | DFS / BFS | `200` |
| 树路径与子树聚合 | DFS | `113`、`2477` |
| 组合、排列、棋盘搜索 | 回溯 | `39`、`46`、`51` |
| 带额外约束的最短路 | Dijkstra / 状态 BFS | `407`、`847` |
| 搜索空间重复 | 记忆化搜索 | `464`、`2850` |

搜索题复盘时，可以固定问四个问题：

- 状态是什么？
- 邻居怎么生成？
- 怎样判重？
- 什么时候可以提前停止或剪枝？

### 本章面试题与追问

- 回溯和普通 DFS 的区别到底在哪里，为什么常说“回溯是带撤销的 DFS”？

## 动态规划

### DP 的核心思想

动态规划的本质是“把重复子问题的答案保存下来”，难点不在代码模板，而在状态设计。

**五个固定问题**

1. `dp` 数组或函数的定义是什么？
2. 状态如何转移？
3. 初始条件和边界条件是什么？
4. 计算顺序是什么？
5. 最终答案落在哪个状态上？

**两种常见视角**

- 自底向上填表：依赖关系清晰、便于做空间优化。
- 自顶向下记忆化：问题表达自然、适合先写递归再补缓存。

**两类依赖模式**

- 局部依赖：`dp[i]` 只看前一两个状态，如 `70. 爬楼梯`、`198. 打家劫舍`、`53. 最大子数组和`。
- 全局扫描：`dp[i]` 需要枚举所有历史状态，如 `300. 最长递增子序列`、`139. 单词拆分`。

把题看成“状态 + 转移图”，比死记“这是哪一类 DP”更可靠。

### 线性 DP 与背包问题

**线性 DP**

- 斐波那契模型：`509. 斐波那契数`、`70. 爬楼梯`、`746. 使用最小花费爬楼梯`。
- 网格路径：`62. 不同路径`、`63. 不同路径 II`、`64. 最小路径和`、`120. 三角形最小路径和`。
- 状态机：`121. 买卖股票的最佳时机`、`122. 买卖股票的最佳时机 II`、`309. 最佳买卖股票时机含冷冻期`。
- 双串与序列：`1143. 最长公共子序列`、`72. 编辑距离`、`97. 交错字符串`、`115. 不同的子序列`。
- 划分型：`132. 分割回文串 II`、`1043. 分隔数组以得到最大和`、`410. 分割数组的最大值`。

**背包问题**

- 0/1 背包：`416. 分割等和子集`、`494. 目标和`、`474. 一和零`。
- 完全背包：`322. 零钱兑换`、`518. 零钱兑换 II`、`279. 完全平方数`、`377. 组合总和 IV`。
- 分组背包：`1155. 掷骰子等于目标和的方法数`。

**最重要的实现差异**

- 0/1 背包一维优化时，容量要倒序遍历。
- 完全背包一维优化时，容量要正序遍历。
- 题目问“最值”“是否可行”“方案数”时，转移写法都不同，不能混套模板。

### 区间 DP、树形 DP、状态压缩 DP

**区间 DP**

- 适合“合并区间”“最后一步切分”的题。
- 代表题：`312. 戳气球`、`877. 石子游戏`、`375. 猜数字大小 II`。
- 典型写法是枚举区间长度，再枚举切分点。

**树形 DP**

- 节点状态依赖子树返回值，适合树上最优解与路径聚合。
- 代表题：`543. 二叉树的直径`、`124. 二叉树中的最大路径和`、`337. 打家劫舍 III`、`834. 树中距离之和`。
- 难点往往不是遍历，而是定义“每个节点返回什么信息”。

**状态压缩 DP**

- 当元素数量很小但子集状态很多时，把集合压成二进制。
- 代表题：`464. 我能赢吗`、`526. 优美的排列`、`698. 划分为 k 个相等的子集`、`2741. 特别的排列`。
- 常和记忆化搜索、位运算一起出现。

### 常见套路总结

| 信号 | DP 类型 | 代表题 |
| --- | --- | --- |
| 只依赖前几个位置 | 线性 DP | `70`、`198`、`746` |
| 二维网格或双串匹配 | 二维 DP | `64`、`1143`、`72` |
| 买卖股票、持有/不持有 | 状态机 DP | `121`、`309` |
| 按区间合并 | 区间 DP | `312`、`375` |
| 子树向上汇总 | 树形 DP | `124`、`337` |
| 小规模集合枚举 | 状压 DP | `464`、`526` |
| 容量限制选物品 | 背包 DP | `416`、`322`、`518` |

复习时不要只背题型名，更要养成这几个习惯：

- 先写出状态含义，再写转移。
- 先画依赖方向，再决定循环顺序。
- 空间优化前先确认不会覆盖掉本轮还需要的旧状态。

## 字符与字符串处理

### 常见字符与字符串处理函数

原稿以 C++ 字符处理函数速查为主，这一章保留最常用的一组，并强调它们在字符串题里的使用边界。

| 函数 | 作用 | 常见用途 |
| --- | --- | --- |
| `isdigit(c)` | 判断是否为数字字符 | 解析数值、过滤输入 |
| `isalpha(c)` | 判断是否为字母 | 单词扫描、词法判断 |
| `isalnum(c)` | 判断是否为字母或数字 | 清洗字符串、回文过滤 |
| `isspace(c)` | 判断是否为空白字符 | 跳过空格、手写解析器 |
| `tolower(c)` | 转小写 | 大小写归一化 |
| `toupper(c)` | 转大写 | 统一比较、格式化输出 |

如果题目不仅涉及单字符分类，还涉及切片、拼接、查找，那么通常还会搭配：

- `substr`：截取子串。
- `find`：查找目标子串或字符。
- `stringstream` 或双指针扫描：处理空格分词。
- `reverse`：原地翻转字符串或区间。

### 使用场景与示例

**场景 1：校验输入是否合法**

- `65. 有效数字` 这类题里，常见做法是结合状态机或分类讨论，判断当前字符是数字、符号、小数点还是指数标记。
- 如果只靠 `isdigit`，往往不够，还要考虑 `+`、`-`、`.`、`e` 的上下文位置。

**场景 2：做大小写无关比较**

- 回文串、单词匹配、日志解析经常要先统一大小写，再比较字符。
- 常见题：`125. 验证回文串`，可配合 `isalnum` 过滤无关字符后，再用双指针判断。

**场景 3：扫描并解析字符串**

- `8. 字符串转换整数 (atoi)`、表达式求值、配置解析都需要“跳过空格 -> 识别符号 -> 读取数字”。
- 这类题适合把字符函数和双指针/有限状态机一起用。

**场景 4：字符串预处理**

- `151. 反转字符串中的单词`、`49. 字母异位词分组`、`14. 最长公共前缀` 这类题，预处理比主算法更关键。
- 常见做法是先去掉多余空格、统一大小写、按规则切词，再进入主逻辑。

### 易错点与延伸阅读

**易错点**

- `isdigit`、`isalpha` 等函数处理的其实是字符分类，不等于“这个字符在当前语义里合法”。
- 直接把 `char` 传给字符分类函数时，若字符集或符号扩展复杂，最好先转成 `unsigned char` 再调用，避免未定义行为风险。
- 大小写转换只适合单字节英文字符语境，多语言文本不能简单套用。
- `isalnum` 会把字母和数字都视为合法字符，做纯数字校验时不能误用。
- 字符串题常见错误不在算法，而在前导空格、符号位、越界、空串这类边界处理。
