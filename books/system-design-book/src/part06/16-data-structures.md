# 第 16 章 数据结构题型速查

> 本部分属于编码补充，用来和系统设计主线互补；更适合作为查漏补缺与面试前速查，不要求线性通读。

## 数组、链表、栈与队列

这类题目的共同点是数据组织方式明确，重点在于识别访问顺序、边界移动方式，以及是否允许原地修改。

**数组与矩阵**

- 原地置换与原地标记：`41. 缺失的第一个正数`、`448. 找到所有数组中消失的数字`、`73. 矩阵置零`。
- 坐标变换与遍历模拟：`48. 旋转图像`、`54. 螺旋矩阵`、`36. 有效的数独`、`31. 下一个排列`。
- 区间与贡献度：`53. 最大子数组和`、`152. 乘积最大子数组`、`907. 子数组的最小值之和`。

**链表**

- 基础反转与局部反转：`206. 反转链表`、`92. 反转链表 II`、`25. K 个一组翻转链表`。
- 快慢指针与结构定位：`141. 环形链表`、`142. 环形链表 II`、`19. 删除链表的倒数第 N 个结点`、`876. 链表的中间结点`。
- 合并、排序与拆分：`21. 合并两个有序链表`、`23. 合并 K 个升序链表`、`148. 排序链表`、`86. 分隔链表`。
- 复杂结构与缓存设计：`138. 随机链表的复制`、`382. 链表随机节点`、`146. LRU 缓存`、`460. LFU 缓存`。

**栈与队列**

- 基础栈模型：括号匹配、表达式处理、递归转迭代。
- 单调栈：`84. 柱状图中最大的矩形`、`496. 下一个更大元素 I`、`739. 每日温度`、`42. 接雨水`、`2866. 美丽塔 II`。
- 双端队列与窗口：`239. 滑动窗口最大值`、`862. 和至少为 K 的最短子数组`、`2810. 故障键盘`。
- 队列分层处理：树的层序遍历、图的 BFS、任务调度等题型。

**做题时先问自己**

- 数据是否要求连续存储，能否通过下标直接映射状态？
- 是否必须保留原顺序，还是只关心相对顺序或分组结果？
- 是否存在“最近更大/更小”“窗口最值”“前驱/后继”这类单调性信号？
- 是否可以通过哑节点、辅助栈、队首队尾维护，把边界操作写干净？

## 哈希表、堆、树与图

这一组题的核心不是“存数据”，而是“如何高效定位、聚合与搜索”。

**哈希表**

- 频次统计与去重：`1. 两数之和`、`49. 字母异位词分组`、`128. 最长连续序列`。
- 前缀和配合哈希：`560. 和为 K 的子数组`、`437. 路径总和 III`。
- 映射复杂对象：克隆图、随机链表复制、缓存设计等。
- 典型信号：题目要求均摊 `O(1)` 查找，或反复判断“是否见过”“出现次数是多少”。

**堆与优先队列**

- Top K 与动态极值：`215. 数组中的第 K 个最大元素`、`347. 前 K 个高频元素`、`295. 数据流的中位数`。
- 多路归并：`23. 合并 K 个升序链表`、`264. 丑数 II`。
- 反悔贪心：`630. 课程表 III`、`871. 最低加油次数`、`502. IPO`。
- 图最短路：`743. 网络延迟时间`、`1976. 到达目的地的方案数`。

**树**

- 遍历与模板：前序、中序、后序、层序遍历。
- 路径与聚合：`112. 路径总和`、`113. 路径总和 II`、`124. 二叉树中的最大路径和`、`543. 二叉树的直径`。
- 二叉搜索树性质：中序有序、范围裁剪、前驱后继。
- Trie 与字典树：`208. 实现 Trie`、`212. 单词搜索 II`，适合多模式前缀匹配。

**图**

- 连通性与遍历：`200. 岛屿数量`、`133. 克隆图`、`2101. 引爆最多的炸弹`。
- 拓扑排序：`207. 课程表`、`210. 课程表 II`。
- 并查集：`547. 省份数量`、`684. 冗余连接`、`1998. 数组的最大公因数排序`。
- 最短路与状态图：`127. 单词接龙`、`407. 接雨水 II`、`2642. 设计图类求最短路径`。

**选择提示**

- “找是否存在、统计频率”优先看哈希。
- “维护动态最小值/最大值/前 K”优先看堆。
- “天然层级结构、父子依赖”优先看树。
- “对象之间有边、有依赖、有可达性”优先建图。

## 常见题型总结

这一章更像一个数据结构选型速查表：

| 信号 | 常用结构 | 代表题 |
| --- | --- | --- |
| 原地修改、数组下标可映射状态 | 数组原地哈希 | `41`、`448`、`73` |
| 最近更大/更小、区间贡献 | 单调栈 | `84`、`739`、`907` |
| 窗口最值、可弹出过期元素 | 双端队列 | `239`、`862` |
| 快速判重、计数、前缀信息 | 哈希表 | `1`、`49`、`560` |
| Top K、动态极值、反悔 | 堆 | `215`、`295`、`630` |
| 指针操作复杂、局部重连 | 链表 + 哑节点 | `19`、`25`、`86` |
| 层级遍历、路径聚合 | 树 | `102`、`124`、`543` |
| 连通性、依赖、最短路 | 图 / 并查集 | `200`、`207`、`743` |

复习时建议按“结构能力”而不是按题号记忆：

- 数组看原地性、连续性、前后缀。
- 链表看指针关系、是否需要前驱、是否适合哑节点。
- 栈和队列看单调性与过期元素处理。
- 树和图看遍历顺序、状态定义和去重方式。

## 本章面试题与追问

- 为什么 `LRU` 需要“哈希表 + 双向链表”，只用其中一种结构为什么不够？
- 单调栈题里，为什么元素出栈时往往意味着“答案已经确定”？
- 用快慢指针找环入口时，为什么相遇后一个指针回到头节点再同步前进即可？
- `Top K` 题什么时候用堆，什么时候用快速选择，什么时候用桶排序？
- DFS、BFS、并查集都能做连通性问题时，面试里该如何解释选型差异？
