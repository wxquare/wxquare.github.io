# 第 17 章 基本算法模式速查

> 本部分属于编码补充，用来和系统设计主线互补；更适合作为查漏补缺与面试前速查，不要求线性通读。

## 双指针、滑动窗口与前缀和

这一组方法都在处理“连续区间”，区别在于维护的信息不同。

**双指针**

- 对撞指针：适合有序数组与区间边界博弈，代表题有 `167. 两数之和 II`、`15. 三数之和`、`11. 盛最多水的容器`、`42. 接雨水`。
- 快慢指针：适合原地压缩与稳定搬运，代表题有 `283. 移动零`、`392. 判断子序列`、`151. 反转字符串中的单词`。
- 中心扩展也可视作双指针特例，常见于 `5. 最长回文子串`、`647. 回文子串`。

**滑动窗口**

- 求最长或最大：窗口合法时持续右扩，例如 `3. 无重复字符的最长子串`、`1004. 最大连续 1 的个数 III`、`2024. 考试的最大困扰度`。
- 求最短或最小：一旦满足条件就尽量收缩左边界，例如 `76. 最小覆盖子串`、`209. 长度最小的子数组`。
- 固定窗口：左右同步移动，更适合做统计量维护，例如 `438. 找到字符串中所有字母异位词`、`1052. 爱生气的书店老板`。

**前缀和**

- 区间和直接查询：`sum[l..r] = pre[r+1] - pre[l]`。
- 配合哈希做计数：`560. 和为 K 的子数组`。
- 配合单调队列解决负数场景：`862. 和至少为 K 的最短子数组`。
- 二维前缀和适合矩阵区域统计，常见于子矩阵和类问题。

**判断标准**

- 区间连续且能在线维护条件，优先考虑窗口。
- 输入有序或能通过排序制造单调性，优先考虑双指针。
- 反复查询区间和、区间计数、区间差值，优先考虑前缀和。

## 二分、排序与贪心

这三类方法都依赖“结构化信息”，只是利用方式不同。

**二分**

- 标准查找：`704. 二分查找`、`34. 在排序数组中查找元素的第一个和最后一个位置`。
- 在旋转数组中找局部单调性：`33. 搜索旋转排序数组`、`153. 寻找旋转排序数组中的最小值`。
- 二分答案：把“能否做到”转成判定函数，例如 `875. 爱吃香蕉的珂珂`、`1011. 在 D 天内送达包裹的能力`、`410. 分割数组的最大值`。

**排序**

- 排序后消除一个维度：`56. 合并区间`、`252. 会议室`、`179. 最大数`。
- 自定义排序：`49. 字母异位词分组`、`179. 最大数`、`2512. 奖励最顶尖的 K 名学生`。
- 排序 + 归并思想：`315. 计算右侧小于当前元素的个数`、`493. 翻转对`。

**贪心**

- 局部最优可推出全局最优：`455. 分发饼干`、`134. 加油站`、`135. 分发糖果`。
- 反悔贪心：`630. 课程表 III`、`502. IPO`、`871. 最低加油次数`。
- 贡献度视角：`2171. 拿出最少数目的魔法豆`、`2731. 移动机器人`。

**面试表达建议**

- 先说明问题是否具备单调性。
- 再说明排序是否改变答案本质，只是帮助更快做决策。
- 最后解释贪心选择为何不会错过最优解，必要时给反证或交换论证。

## 递归与回溯基础

递归适合表达“问题拆成同类子问题”，回溯是在递归基础上显式撤销选择。

**递归框架**

- 明确函数含义、终止条件、子问题返回值。
- 树题常用“返回子树信息”的递归。
- 分治题常用“拆分后合并”的递归，如归并排序、快速幂、快速选择。

**回溯框架**

- 路径 `path`：当前已经做出的选择。
- 选择列表：当前位置还能尝试什么。
- 结束条件：何时收集答案或提前返回。
- 撤销选择：恢复现场，避免污染其他分支。

**三类高频模型**

- 组合：用 `start` 控制“之后还能选谁”，代表题 `77. 组合`、`39. 组合总和`、`216. 组合总和 III`。
- 排列：用 `used` 控制“哪些元素已经用过”，代表题 `46. 全排列`、`47. 全排列 II`。
- 分割：枚举切分点并维护合法性，代表题 `131. 分割回文串`、`LCR 086. 分割回文串`。

**常见剪枝**

- 排序后剪枝：重复元素跳过、超过目标直接终止。
- 上下界剪枝：剩余元素数量不足，或当前和已经不可能达到目标。
- 记忆化：相同状态重复访问时直接复用结果。

## 常见题型总结

| 信号 | 方法 | 代表题 |
| --- | --- | --- |
| 连续区间、窗口可维护 | 滑动窗口 | `3`、`76`、`209` |
| 有序数组、两端逼近 | 双指针 | `167`、`15`、`11` |
| 高频区间统计 | 前缀和 | `560`、`862` |
| 答案具有单调性 | 二分答案 | `875`、`1011`、`410` |
| 先排序再做决策更自然 | 排序 + 贪心 | `56`、`252`、`455` |
| 枚举所有方案 | 回溯 | `39`、`46`、`131` |

把这些方法连起来理解会更稳：

- 双指针与窗口本质上都在维护一个区间。
- 前缀和负责把“区间信息”提取成常数时间查询。
- 二分和贪心通常依赖排序或单调性来缩小搜索空间。
- 回溯负责“全搜”，剪枝负责“少搜”。

## 本章面试题与追问

- 滑动窗口和双指针的边界区别是什么，为什么两者经常被混用？
- 二分答案题里，如何快速判断题目是否满足“可行解单调”？
- 贪心题中如果无法严格证明，面试时该如何稳妥表述？
- 回溯去重时，为什么常见写法是“排序后跳过同层重复元素”？
- `560` 和 `862` 都和前缀和有关，为什么一个用哈希，一个要加单调队列？
