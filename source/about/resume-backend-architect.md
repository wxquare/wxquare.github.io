作为核心项目组成员，端到端参与 Shopee 数字电商 C 端购买主链路建设，支撑多市场，覆盖酒店、机票、电影票、账单缴费、充值、本地生活等数十种数字商品，承载百万级日订单和核心链路峰值 2,000+ QPS；在供应商履约与平台自营库存并存的复杂场景下，重点负责价格/库存一致性治理与交易模型建设。

- 针对多品类流程差异大、资源确认型商品外部不确定性强的问题，抽象“交易阶段 + 交易前上下文 + 订单快照”模型，沉淀 Booking → Order → Payment → Fulfillment / Refund 交易编排；导购阶段输出商品/价格/可售上下文，Booking 阶段处理供应商查价查库存、占座/锁房/订票确认和超时释放，Order 阶段固化商品、价格、履约和售后规则快照，支付/履约/退款回调通过状态机、失败补偿和对账推进订单闭环。
- 针对价格逻辑分散、列表价/详情价/订单价/支付价口径不一致的问题，主导统一计价中心建设，将商品基础价、购买数量、营销优惠、手续费、券抵扣等输入收敛为场景化计算模型；通过订单/支付双快照、空跑比对、灰度迁移和异常拦截，项目上线后未造成资损事故。
- 针对搜索、详情、试算和下单前确认等高频路径在性能与准确性上的冲突，按“列表快、详情准、创单安全”分层治理价格/库存新鲜度；通过场景化 TTL、多级缓存、批量聚合、并发查询、热点保护和降级契约，支撑日均 5000 万+ 调用，关键接口 P99 从 500ms 优化至 <200ms。
- 针对支付、履约、退款回调存在重试、乱序和部分失败的问题，抽象支付单、订单、订单项、履约单、退款单模型，支持单商品、多商品、合并支付、分单履约和部分退款；通过幂等、防重、乐观锁、延时队列和扫表补偿降低最终一致性风险。

**商品供给、库存与运营平台化**

围绕“供应商/商家供给如何稳定转化为 C 端可售商品”，建设多品类商品供给与库存平台，覆盖商品建模、库存管理、价格维护、审核发布、供应商同步、批量运营和审计追踪。支撑酒店、机票、船票、电影票、电子券、话费、礼品卡等 10+ 数字商品品类，解决早期品类模型割裂、供给链路重复、库存口径不统一和运营效率低的问题。

- 抽象 item / sku / offer / 属性 / 价格 / 库存 / 审核 / 发布等通用供给模型，通过动静分离、MTI + EAV、策略模式和统一状态机承载品类差异，代码复用率从 <10% 提升至 90%+，新品类接入从 2 周缩短至 2 天。

- 将券码库存、数量库存、时间维度库存、供应商实时库存抽象为统一“库存管理类型 × 库存单元类型”模型，支撑预占、扣减、释放、补货、对账和活动独立库存；秒杀峰值 20,000 QPS，日均 200 万订单，数量制预订 P99 <30ms。
- 沉淀统一批量运营框架，复用任务表、流式 Excel 解析、Worker Pool、分批 SQL、结果文件和审计链路，支撑万级 SKU 调价、设库、券码导入和批量上下架，新增批量能力开发成本降低 7 倍，运营操作从小时级压缩到分钟级。
