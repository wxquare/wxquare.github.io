- **多品类购买链路统一**：针对充值/账单、酒店/机票、电影票、券类商品在搜索展示、详情聚合、资源确认、支付履约和退款规则上的差异，将 C 端链路拆分为 Search / Detail / Cart / Booking / Order / Checkout / Payment / Fulfillment / Refund 标准阶段；Search 统一商品召回、筛选排序和导购价格，Detail 聚合商品、实时价格/库存、营销和履约规则，Booking 承接供应商查价查库存、锁房/占座/订票确认和超时释放，Order 固化商品/价格/资源/履约/售后快照，Payment / Fulfillment / Refund 通过状态机、幂等和补偿任务推进闭环，降低新品类接入和交易一致性治理成本。
- **计价中心与金额一致性**：针对商品、营销、订单、支付多处价格逻辑分散，以及列表价、详情价、订单价、收银台支付价口径不一致的问题，主导统一计价中心建设；按 PDP / Cart / Order / Checkout / Payment 拆分场景化计算能力，沉淀 Base Price → Promotion → Fee → Discount 四层模型和品类 Calculator 扩展机制，通过 PriceBreakdown、订单/支付快照、空跑比对、灰度迁移和异常拦截保障价格可解释与金额一致，项目上线后未造成资损事故。
- **首页入口与导购性能治理**：针对首页 Entrance/Group 配置大、访问集中，以及搜索、详情、试算和下单前确认等高频读路径在性能与准确性上的冲突，按“首页稳、列表快、详情准、创单安全”分层治理数据新鲜度；通过配置瘦身、Redis 热 Key 分散、本地缓存、CDN 降级、多级缓存、批量聚合、并发查询和热点保护，支撑日均 5000 万+ 调用，关键接口 P99 从 500ms 优化至 <200ms。
- **状态机与最终一致性**：针对支付、履约、退款回调存在重试、乱序和部分失败的问题，抽象支付单、订单、订单项、履约单、退款单模型，支持单商品、多商品、合并支付、分单履约和部分退款；通过幂等、防重、乐观锁、延时队列和扫表补偿降低最终一致性风险。

**商品供给、库存与运营平台化**

围绕“供应商/商家供给如何稳定转化为 C 端可售商品”，建设多品类商品供给与库存平台，覆盖商品建模、库存管理、价格维护、审核发布、供应商同步、批量运营和审计追踪。支撑酒店、机票、船票、电影票、电子券、话费、礼品卡等 10+ 数字商品品类，解决早期品类模型割裂、供给链路重复、库存口径不统一和运营效率低的问题。

- 抽象 item / sku / offer / 属性 / 价格 / 库存 / 审核 / 发布等通用供给模型，通过动静分离、MTI + EAV、策略模式和统一状态机承载品类差异，代码复用率从 <10% 提升至 90%+，新品类接入从 2 周缩短至 2 天。

- 将券码库存、数量库存、时间维度库存、供应商实时库存抽象为统一“库存管理类型 × 库存单元类型”模型，支撑预占、扣减、释放、补货、对账和活动独立库存；秒杀峰值 20,000 QPS，日均 200 万订单，数量制预订 P99 <30ms。
- 沉淀统一批量运营框架，复用任务表、流式 Excel 解析、Worker Pool、分批 SQL、结果文件和审计链路，支撑万级 SKU 调价、设库、券码导入和批量上下架，新增批量能力开发成本降低 7 倍，运营操作从小时级压缩到分钟级。
