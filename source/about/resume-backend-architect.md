**演进路径与核心贡献：**

- **初期阶段**：以运营手动表单（Local）创建商品为核心，快速支撑 Topup / Bill 等业务上线。

- **多源供给收拢阶段**：设计统一的多源供给接入架构，整合供应商 Pull / Push 同步、ISV API 推送、商家自助、Excel 批量导入等多种方式；沉淀标准商品模型、供应商 Mapping、字段清洗、数据校验和发布治理能力，支撑业务向 Hotel / Ferry / Deal 等多品类扩展。发布环节采用 Saga 编排，确保商品、SKU、库存、搜索索引和缓存的一致性与可恢复性。

**2. C 端核心交易平台与交易安全**

- **交易链路标准化**：将多品类交易链路抽象为 Search / Detail / Cart / Checkout / Order / Payment / Fulfillment / Refund 标准阶段，明确导购弱一致、详情近实时、结算强校验、订单快照和售后事实治理的领域边界，降低新品类交易接入复杂度。
- **订单快照机制**：设计结构化订单快照，在创单时固化商品、价格、库存 / 资源、履约规则和退款规则，确保支付、履约、退款和客服仲裁基于创单时事实，避免依赖最新商品状态解释历史订单。
- **长链路最终一致性**：设计“预占凭证 + 本地事务落订单 + Outbox 异步 Confirm / Cancel”的改良 Saga 模式，结合幂等键、防重令牌、乐观锁、延时队列和定时补偿任务，处理支付回调乱序、供应商履约超时、支付超时取消和退款回补失败等一致性问题。

- **交易安全治理**：通过 Checkout 实时强校验、库存预占、券码锁定、版本校验、超时释放和售后事实治理，降低价格 / 库存不一致、超卖、无券可发、退款回补失败等资损风险。
