# 在Cursor中使用AI辅助博客维护

本文档介绍如何在Cursor IDE中使用AI来辅助维护这个Hexo博客项目。

## 🎯 核心区别

| 功能 | Claude Code | Cursor | 说明 |
|------|-------------|--------|------|
| 配置文件 | CLAUDE.md | .cursorrules | Cursor使用.cursorrules |
| Skills | ✅ 原生支持 | ⚠️ 需要用自然语言 | 通过对话实现类似功能 |
| 自动化 | Hooks + MCP | 🔧 需要脚本 | 可以用Composer或手动 |
| 代码生成 | ✅ | ✅ | 都支持 |

- ✅ `.cursorrules` - Cursor AI规则文件（已创建）
- ✅ 文章模板（`.claude/templates/`）
- ✅ 检查脚本（`scripts/pre-commit-check.sh`）

## 🚀 在Cursor中的使用方法

### 1. 创建新文章

**在Cursor中，使用Cmd+K或Cmd+L，然后输入：**

```
请帮我创建一篇新的博客文章：
- 标题：Claude Code使用指南
- 分类：AI
- 标签：claude-code, ai-tools, productivity
```

**AI会：**
1. 询问是否需要子分类
2. 生成合适的文件名
3. 创建文件到正确的目录
4. 添加完整的Front Matter
5. 提供文章结构模板

### 2. 审查文章质量

**选中文章内容，然后Cmd+K：**

```
请审查这篇文章，检查：
1. Front Matter完整性
2. 内容结构
3. 代码质量
4. 格式规范
5. 技术准确性
6. SEO优化
7. 可读性
8. 链接有效性
```

**或者更简单：**
```
按照项目规范审查这篇文章
```

### 3. 生成文章摘要

**选中文章，Cmd+K：**

```
为这篇文章生成：
1. 一句话摘要（20-30字）
2. 段落摘要（100-150字）
3. 详细摘要（300-500字）
4. SEO关键词建议
5. 相关标签建议
```

### 4. 优化代码示例

**选中代码块，Cmd+K：**

```
优化这段代码：
1. 添加注释
2. 改进命名
3. 添加错误处理
4. 确保可运行
```

### 5. 检查格式规范

**选中段落，Cmd+K：**

```
检查并修复：
1. 中英文之间的空格
2. 标点符号使用
3. 代码块语言标注
4. 链接格式
```

## 🎨 Cursor特有功能

### 1. Composer（Cmd+I）
用于多文件编辑，适合：
- 批量修改多篇文章
- 重构文章结构
- 统一格式规范

**示例：**
```
Cmd+I
输入：检查所有AI分类的文章，确保标签使用统一
```

### 2. Chat（Cmd+L）
用于对话和规划，适合：
- 讨论文章思路
- 获取技术建议
- 规划文章结构

### 3. Inline Edit（Cmd+K）
用于快速编辑，适合：
- 修改选中的文本
- 生成代码示例
- 格式化内容

## 💡 最佳实践

### 1. 利用.cursorrules
- Cursor会自动读取`.cursorrules`
- 不需要每次都重复说明规范
- 直接说"按照项目规范"即可

### 2. 使用@符号引用
```
@.claude/templates/tech-tutorial.md 帮我创建类似的文章
@source/_posts/AI/claude-code-guide-summary.md 参考这篇文章的风格
```

### 3. 分步骤执行
- 不要一次性要求太多
- 每完成一步验证结果
- 逐步完善文章

### 4. 保存常用提示词
- 在Cursor中可以保存常用的提示词
- 创建自己的提示词库

### 5. 结合终端命令
```
Cmd+L
输入：帮我生成一个命令来统计各分类的文章数量
```

## ❓ 常见问题

**Q: .cursorrules不生效？**
A: 重启Cursor，或者在设置中检查是否启用了Rules功能。

**Q: AI不理解项目规范？**
A: 在提示词中明确说"按照.cursorrules中的规范"。

**Q: 如何让AI记住上下文？**
A: 使用Cmd+L的聊天功能，保持对话连续性。

**Q: 可以自动化发布吗？**
A: 可以，但需要手动运行脚本或配置Git hooks。

**Q: 与Claude Code相比的优势？**
A: Cursor集成在IDE中，编辑更直观，代码补全更强大。
