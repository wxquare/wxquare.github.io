# /review-post - 博客文章质量审查

对博客文章进行全面的质量检查，确保内容符合规范和最佳实践。

## 检查清单

### 1. Front Matter完整性检查
- [ ] title字段存在且非空
- [ ] date字段存在且格式正确（YYYY-MM-DD）
- [ ] categories字段存在且符合规范
- [ ] tags字段存在且至少有2个标签
- [ ] 所有字段格式正确（YAML语法）

### 2. 内容结构检查
- [ ] 文章有清晰的引言部分
- [ ] 标题层级正确（# 只用一次作为文章标题）
- [ ] 使用## 和### 组织内容结构
- [ ] 有明确的总结部分
- [ ] 段落长度适中，避免过长段落

### 3. 代码质量检查
- [ ] 所有代码块都指定了语言（```python, ```go等）
- [ ] 代码示例完整可运行
- [ ] 复杂代码有必要的注释
- [ ] 代码格式规范（缩进、命名等）
- [ ] 提供了依赖版本信息（如需要）

### 4. 格式规范检查
- [ ] 中英文之间有空格
- [ ] 中文使用中文标点，英文使用英文标点
- [ ] 列表格式统一（- 或 1. ）
- [ ] 链接格式正确 [文字](URL)
- [ ] 图片路径正确且可访问

### 5. 技术准确性检查
- [ ] 技术术语使用一致
- [ ] 概念解释准确
- [ ] 代码逻辑正确
- [ ] 引用来源标注清楚
- [ ] 没有过时或错误的信息

### 6. SEO优化检查
- [ ] 标题包含关键词
- [ ] 至少3个相关标签
- [ ] 文章长度适中（建议800字以上）
- [ ] 有内部链接到相关文章
- [ ] 图片有alt属性（如使用图片）

### 7. 可读性检查
- [ ] 语言流畅，逻辑清晰
- [ ] 避免过于专业的术语（或有解释）
- [ ] 有适当的示例和图表
- [ ] 重点内容有突出标记（加粗、引用等）
- [ ] 避免冗长的句子

### 8. 链接检查
- [ ] 所有外部链接可访问
- [ ] 内部链接路径正确
- [ ] 没有死链接
- [ ] 引用的GitHub仓库存在

## 审查流程

1. **读取文章内容**
   - 解析Front Matter
   - 分析文章结构
   - 提取代码块和链接

2. **执行检查**
   - 按照检查清单逐项检查
   - 记录发现的问题
   - 评估问题严重程度（严重/一般/建议）

3. **生成报告**
   - 列出所有发现的问题
   - 按严重程度分类
   - 提供具体的修改建议
   - 给出总体评分

4. **提供改进建议**
   - 内容改进建议
   - SEO优化建议
   - 相关文章推荐（可添加内部链接）

## 输出格式

```
📝 文章审查报告：[文章标题]

✅ 通过项：X/Y

❌ 严重问题：
1. [问题描述] - [修改建议]

⚠️ 一般问题：
1. [问题描述] - [修改建议]

💡 优化建议：
1. [建议内容]

🔗 推荐相关文章：
- [文章1]
- [文章2]

总体评分：★★★★☆ (4/5)
```

## 使用示例

```
/review-post source/_posts/AI/Codex-guide-summary.md
```

## 注意事项

- 技术文章重点检查准确性
- 教程类文章重点检查完整性
- 思考类文章重点检查逻辑性
- 根据文章类型调整检查重点
