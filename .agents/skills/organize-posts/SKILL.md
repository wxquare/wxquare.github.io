# /organize-posts - 整理博客文章结构

扫描所有博客文章，分析分类结构，提供整理建议。

## 功能

### 1. 扫描文章结构
- 统计各分类的文章数量
- 识别分类不合理的文章
- 发现命名不规范的文件
- 检测重复或相似主题的文章

### 2. 分析建议
- **需要移动的文章**
  - AI相关但未放在AI目录
  - 应该归入子目录的文章
  - 分类错误的文章

- **需要合并的文章**
  - 主题高度重复
  - 内容可以整合
  - 系列文章可以合并

- **需要拆分的文章**
  - 文章过长（超过5000字）
  - 包含多个独立主题
  - 可以形成系列文章

### 3. 生成索引
- 按分类生成文章索引
- 按标签生成文章索引
- 按时间生成文章索引
- 生成文章关系图

### 4. 识别需要更新的文章
- 超过1年未更新的技术文章
- 引用了过时技术的文章
- 链接失效的文章
- 内容不完整的文章

## 执行步骤

1. **扫描所有文章**
```bash
find source/_posts -name "*.md" -type f
```

2. **解析Front Matter**
   - 提取分类、标签、日期信息
   - 分析文章元数据

3. **分析文章内容**
   - 统计字数
   - 提取关键词
   - 识别主题

4. **生成整理报告**

## 使用场景

- 定期整理（建议每月一次）
- 新增多篇文章后
- 重构博客结构前
- 准备归档旧文章时

## 注意事项

- 移动文章前先备份
- 更新内部链接
- 检查图片路径
- 重新生成索引
