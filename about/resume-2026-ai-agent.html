◇2021.07 - 至今（5 年）　Shopee / Digital Purchase 数字电商平台 / Listing & Order 组
1. AI 告警处理 Agent（DoD Agent）— 从 0 到 1 设计与落地
  • 架构设计：设计状态机 + ReACT 混合架构的智能告警处理 Agent；状态机控制告警生命周期（NEW → ANALYZING → AUTO_RESOLVING / WAITING_CONFIRM / ESCALATED → RESOLVED），ReACT 引擎负责智能分析和工具调用，实现流程可控且保留 LLM 推理灵活性。
  • RAG 知识检索 + 工具调用：接入 Confluence / Runbook / 历史案例构建 RAG 知识库（向量数据库 Chroma），注册 10+ MCP 工具（Prometheus 查指标、K8s 查 Pod 状态、日志检索、Jira 创建工单、Seatalk 通知），Agent 通过 ReACT 循环自主选择工具链完成诊断。
  • 分级决策与渐进式学习：四级风险评估（低风险自动处理 / 中风险建议+确认 / 高风险人工确认 / 严重立即升级），策略可配置；基于历史处理反馈持续优化 Prompt 和决策阈值。
  • 成果：自动处理 65%+ 告警，告警响应时间从 15 分钟 → < 2 分钟，值班人员工作量降低 60%；告警噪音减少 70%（智能聚合 + 去重）。
2. 电商核心系统架构（商品 / 库存 / 价格）
  • 统一商品运营平台：策略模式 + 统一状态机支撑 10+ 品类，代码复用率 <10% → 90%+，新品类接入 2 周 → 2 天；统一批量操作框架（流式解析 + Worker Pool + 分批 SQL），万级 SKU 操作从数小时降至分钟级。
  • 多品类统一库存系统：二维分类模型（管理类型 × 单元类型）统一异构库存；Redis Lua 原子操作支撑秒杀 20,000 QPS，日均 200 万订单；库存恒等式 + 小时级自动对账保障一致性。
  • 统一价格计算引擎：DDD + 四层计价模型收敛分散逻辑为统一引擎（日均 5000 万+ 调用）；双快照机制防资损，空跑比对 + 三阶段灰度安全迁移 10+ 品类，资损 3-5 次/年 → 0 次；P99 500ms → <200ms。
3. OTA 品类从 0 到 1（Hotel / Ferry / Flight）
  • 连续主导 3 个 OTA 品类全链路搭建（供应商对接 → 商品建模 → 搜索导购 → 下单履约 → 逆向退款），单品类 2-3 个月交付，第二个品类起复用度 60%+；多供应商聚合架构 + ES 双索引，搜索 RT 降低 50%。
◇2018.09 - 2021.07（3 年）　腾讯 / 微信广告部 / 引擎策略组
